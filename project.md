# Codebase Reorganization Project

Last updated: 2026-04-13

## Goal

Reorganize the codebase in small, test-gated phases without breaking the app.

## Working Rules

1. Keep the public contract stable while reorganizing.
2. Make structural changes in small phases.
3. Run the required test gate after every phase.
4. Only continue to the next phase after the current phase passes.
5. Update this file when a phase starts, finishes, or changes scope.
6. Keep compatibility wrappers until the final cleanup phase.
7. Temporary app downtime is acceptable during an active refactor batch if it speeds the migration, but every batch must still pass its required gates before the next batch begins.

## Test Gates

Note:
- In this environment, some temp-directory dependent tests need to run outside the workspace sandbox. When that happens, rerun the same pytest command with broader filesystem access instead of weakening the gate.
- On this Windows setup, `python tests/test_watchlist_e2e.py` may also need UTF-8 console output enabled, for example with `PYTHONIOENCODING=utf-8`, so the script can print its status lines cleanly.
- The app is still pre-launch and not yet customer-facing, so temporary downtime during larger refactor batches is acceptable while the required test gates remain the batch boundary before moving on.

- Run targeted tests immediately after each deletion batch, then the required phase gates at the batch boundary.
- Restore the batch from the local backup archive if the targeted checks or required gates fail.
